Get the IP address of a machine

ifdata -pN eth0 
192.168.1.0

No junk, no pipe, one command, no subcommand - KISS

Alternatives

/sbin/ifconfig -a | awk '/(cast)/ { print $2 }' | cut -d':' -f2 | head -1
ip -o -4 addr show | awk -F '[ /]+' '/global/ {print $4}' 
127.0.0.1
10.80.86.3

Why use many different utilities all piped together, when you only need two?

ifconfig -a | awk '/Bcast/{print $2}' | cut -c 5-19
ifconfig -a | awk '/Bcast/{print $2}' | cut -c 5-19

What others think

That is incorrect, -pN will print out the network address, which is not the equivelent of the host's IP, for that you need ifdata -pa. Also, moreutils (from which ifdata belongs) isn't particulary common, and certainly less common than iproute2 .. or (I dread to mention) net-tools.
