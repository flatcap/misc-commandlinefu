find all c and cpp files except the ones in the unit-test and android subdirectories

find . ! -regex '.*/\(unit-test\|android\)/.*' \( -name '*.c' -o -name '*.cpp' \)
find all c and cpp files except the ones in the unit-test and android subdirectories

Find files matching multiple descriptions (*.c and *.cpp) excluding multiple directories (unit-test and android).

Alternatives

ls **/*.c(|pp)~(unit-test|android)/*
find all c and cpp files except the ones in the unit-test and android subdirectories

works only in zsh, requires EXTENDED_GLOB to be set.

find . -name unit-test -o -name '*.c' -o -name '*.cpp' | egrep -v "unit-test|android"

Very useful for finding all the source code that should be compiled.
